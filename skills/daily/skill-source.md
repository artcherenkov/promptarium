# Запись дейлика

## Контекст

Пользователь ведёт лог ежедневной деятельности для:
- Отчётности по проектам
- Подготовки к грейдированию (достижения + факты)
- Фиксации артефактов своей работы

Ввод — поток сознания (текст или голосовой транскрипт). Твоя задача — структурировать.

**Голосовой ввод:** пользователь часто диктует через speech-to-text. Транскрипт может содержать артефакты: искажённые слова, неправильная транслитерация (например "эфиду" вместо "URL", "хуткейн" вместо "hotkey"). Интерпретируй слова в контексте. Если не можешь понять, что имелось в виду — **спроси**, не угадывай.

## Алгоритм

### 1. Прочитай текущий дейлик

Файл: `/Users/acherenkov/Obsidian/Cian/Дейлики/{сегодняшняя дата}.md`

### 2. Выдай summary

**ПЕРЕД любыми записями** — выдай краткое структурированное summary того, что ты понял из потока сознания:
- Что сделано / что в плане
- К какому проекту относится
- Ключевые детали (названия, ID, технические решения)

Дождись подтверждения или корректировок от пользователя. Только после этого переходи к записи.

### 3. Задай уточняющие вопросы

**ОБЯЗАТЕЛЬНО спрашивай через AskUserQuestion если:**
- Непонятно, это план или уже сделано
- Непонятно, к какому проекту относится
- Непонятно, был ли результат успешным
- Есть любая неясность — спроси

Не додумывай. Всегда уточняй.

### 4. Классифицируй информацию

Из потока сознания выдели:

| Категория | Куда записать |
|-----------|---------------|
| План на сегодня | Дейлик сегодня → секция `## План` (чекбоксы `- [ ]`) |
| План на другой день | Дейлик **того дня** → секция `## План` (создать файл если нет) |
| Выполненные задачи | Дейлик → отметить `- [x]` или добавить в `## Сделано` |
| Прогресс по проекту | Страница проекта → секция `## Таймлайн` |
| Достижение | Где уместно + тег `#молодец` |
| Блокер/сложность | Страница проекта → `## Таймлайн` или `## Блокеры` |
| Инсайт/изменение требований | Страница проекта → `## Таймлайн` |

### 5. Определи достижения (#молодец)

Достижение = влияние на бизнес. Критерии:
- Помог коллеге разобраться
- Затащил новую технологию
- Предложил новый подход
- Уложился в сроки проекта
- Высокое качество кода (нет багов, нет замечаний на ревью)
- Участвовал в развитии важных для бизнеса продуктов

Не нужны точные цифры. Важна суть влияния.

### 6. Обнови файлы

**Дейлик** — структура (без заголовка `# DD Month YYYY` — Obsidian отображает название файла как заголовок):
```markdown
## План

- [ ] Задача 1
- [ ] Задача 2

## Сделано

- [x] Выполненная задача
- Заметка о дне
```

**Таймлайн проекта** — формат записи:
```markdown
## Таймлайн

*29 января* — Описание события
```

Формат даты: `*D месяц*` — курсив, без ведущего нуля (например: `*3 февраля*`, `*12 марта*`).

Примеры записей таймлайна:
- `*29 января* — Закрыл баг CD-261415`
- `*2 февраля* — Изменились требования: теперь нужна поддержка X`
- `*5 февраля* — Провёл ревью PR #7, нашёл проблему в валидации #молодец`

## Связь с проектами

Активные проекты лежат в `Проекты/Q1-Q4/`. Если пользователь упоминает проект:
1. Найди соответствующий файл
2. Добавь запись в `## Таймлайн` (создай секцию если нет)
3. Свяжи дейлик и проект через `[[wikilinks]]`

## Принципы

- **Всегда уточняй** — не додумывай за пользователя
- **Пропускай через призму** — "как это повлияло на бизнес?"
- **Фиксируй артефакты** — ссылки на PR, задачи, документы
- **Отмечай достижения** — тег #молодец где уместно
- **Замечай масштаб** — если из потока сознания вырисовывается полноценный проект (не просто задача дня), предложи переключиться на скилл `project-report` для создания полного отчёта
